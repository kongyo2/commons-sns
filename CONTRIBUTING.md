# Commonsへの参加

Commonsでは、プログラムを書かない参加もコントリビューションとして扱います。

## 参加方法

- 不具合を再現し、環境と手順を報告する
- 新機能を提案し、必要な理由を説明する
- 仕様やUIの議論に参加する
- アクセシビリティや文章表現をレビューする
- 翻訳、テスト、ドキュメントを改善する
- Issueに紐づいたPull Requestを送る

## 機能提案の流れ

1. 既存のIssueとDiscussionに同じ提案がないか確認します。
2. 「誰の、どんな問題を解決するか」を書いて提案します。
3. メンテナーが「検討中」「設計募集」「実装可能」「今回は見送り」に分類します。
4. 仕様が固まったものだけを実装します。
5. レビューとテストを通過した変更を段階的に公開します。

要望の数だけで採用は決まりません。安全性、プライバシー、操作の一貫性、運用コストも判断材料になります。

## Pull Request

- 大きな変更は、実装前にIssueで方向性を確認してください。
- 1つのPull Requestには1つの目的だけを含めてください。
- UI変更には、PCとモバイルのスクリーンショットを添付してください。
- 挙動の追加・変更にはテストを追加してください。
- 破壊的変更やデータ移行がある場合は明記してください。

初参加者向けの課題には `good first issue`、実装者を募集する課題には `help wanted` を付けます。
